# Address Search Component

## Address Search Props

|        Prop        |                                                                       Type                                                                       | Default | Optional |
| :----------------: | :----------------------------------------------------------------------------------------------------------------------------------------------: | :-----: | :------: |
|    getListTitle    |                                         (selected: google.maps.places.AutocompletePrediction) => string                                          |    -    |  false   |
|         id         |                                                                      string                                                                      |    -    |  false   |
|      onChange      | ([suggestion](https://developers.google.com/maps/documentation/javascript/reference/places-autocomplete-service#AutocompletePrediction)) => void |    -    |  false   |
|    placeholder     |                                                                      string                                                                      |    -    |   true   |
|      selected      |                                                    google.maps.places.AutocompletePrediction                                                     |    -    |  false   |
| variablesClassName |                                                                      string                                                                      |    -    |   true   |

**onChange**

- The onChange event returns a [suggestion](https://developers.google.com/maps/documentation/javascript/reference/places-autocomplete-service#AutocompletePrediction) object, you will be able to do whatever you want with the selected address.

## Getting Started

We created a **MapsProvider** component to set up your _GOOGLE_API_KEY_ once only, it helps in case you need to use one or more AddressSearch component in your application, for it the MapsProvider has to wrapper your AddressSearch components.

[Provider Concept](https://react-redux.js.org/api/provider)

```javascript
import React, { Component, useState } from 'react';
import { AddressSearch, MapsProvider } from '@codelittinc/agnostic-design-system';

class SimpleAddressSearch extends Component {
  const [value, setValue] = useState();

  render() {
    return (
      <MapsProvider apiKey={GOOGLE_API_KEY}>
        <AddressSearch
          selected={value}
          onChange={value => {
            setValue(value);
          }}
          placeholder='Enter Address'
          id='ads-address-search'
        />
      </MapsProvider>
    );
  }
}

export default SimpleAddressSearch;
```

# Customazing styles

For the [AddressSearch](https://design-system.codelitt.dev/?path=/story/components-address-search--default) component we use the following [naming convention](https://design-system.codelitt.dev/?path=/story/welcome-customizing-styles--page) to configure the CSS variables:

**--address-search-[attribute]**

- Attribute - background-color, border, font, etc (required)

**Important:** The AddressSearch component is based on the editable [Dropdown](https://design-system.codelitt.dev/?path=/story/components-dropdown--editable-dropdown) component and you will be able to use its variables to customize this component.

1. Create the css class using our variables

```css
.my-address-css {
  /* Component + Attribute */
  --address-search-border-radius: 1px solid grey;

  /* Component + Attribute */
  --input-background-color: white;

  /* Component + Property + Attribute */
  --input-placeholder-color: grey;

  /* Component + Attribute */
  --list-border: 1px solid #a2cee1;

  /* Component + Attribute + State*/
  --list-item-border-bottom-hover: 1px solid red;
}
```

2. Pass the css class through its `variablesClassName` property

`<AddressSearch variablesClassName={styles['my-address-css']} />`

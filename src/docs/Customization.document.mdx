# Naming Convention

To customize out components we need to follow a specific naming convention to promote predictability and consistency. All of our CSS custom properties adhere to the following convention:

**_--[COMPONENT]-[PROPERTY]?-[ATTRIBUTE]-[STATE]?_**

- COMPONENT - Name of the component **(required)**
- PROPERTY â€” Component property - Button icon as example **(optional)**
- ATTRIBUTE - border, font, etc **(required)**
- STATE - hover, focus, active **(optional)**

## Usage Combinations and Examples

```css
.my-component-css {
  /* Component + Attribute */
  --component-background: red;

  /* Component + Attribute + State */
  --component-background-hover: blue;

  /* Component + Property + Attribute */
  --component-icon-fill: black;

  /* Component + Property + Attribute + State */
  --component-icon-fill-hover: red;
}
```

### How to apply on our components

All of our components have the property `variablesClassName` where it is possible to send your CSS customization.

**Important**: `You will always need to import your CSS file after our CSS file`

```javascript
import '@codelittinc/agnostic-design-system/dist/main.css';
import styles from 'your_css.css';
```

### Example

1. Using the [Button](https://design-system.codelitt.dev/?path=/story/components-button--neutral) component

```css
.my-button-css {
  /* Component + Attribute */
  --button-background-color: #f2eff8;

  /* Component(button) + Attribute(background-color) + State(hover) */
  --button-background-color-hover: #898893;

  /* Component(button) + Attribute(background-color) + State(hover) */
  --button-color: #002c4c;

  /* Component(button) + Attribute(background-color) + State(hover) */
  --button-color-hover: #ffffff;
}
```

```javascript
import React, { Component } from 'react';
import { Button } from '@codelittinc/agnostic-design-system';
import styles from './my-button.css';

class CustomizedButton extends Component {
  render() {
    return <Button variablesClassName={styles['my-button-css']} />;
  }
}

export default CustomizedButton;
```

2. Using the [Input](https://design-system.codelitt.dev/?path=/story/components-input--default) component where we could add **Input properties** to the CSS class

```css
.my-input-class {
  /* Component(input) + Attribute(background-color) */
  --input-background-color: #ffffff;

  /* Component(input) + Attribute(border) + State(hover) */
  --input-border-hover: 1px solid grey;

  /* Component(input) + Property(description) + Attribute(color) */
  --input-description-color: green;
}
```

```javascript
import React, { Component } from 'react';
import { Input } from '@codelittinc/agnostic-design-system';
import styles from './my-input.css';

class CustomizedInput extends Component {
  render() {
    return (
      <Input
        variablesClassName={styles['my-input-class']}
        label='Simple Input'
        description='Simple description'
      />
    );
  }
}

export default CustomizedInput;
```
